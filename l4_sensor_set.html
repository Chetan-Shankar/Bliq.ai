What Changes for a Robotaxi

The set above is a privately-owned-car L4 reference. Remove the human owner and the styling constraints, and the sensor philosophy shifts.

A consumer L4 car has to look like a car — sensors hide in the grille, mirrors, and roofline, and the bill of materials has to survive a showroom price tag. A robotaxi has neither constraint: it is a commercial asset amortised over hundreds of thousands of paid miles, so it can carry more sensors, mount them where physics prefers rather than where design permits, and lean hard on a roof position no production car would accept.

The single biggest visual difference is the move to a roof-mounted 360° sensor dome — a spinning or solid-state long-range lidar plus a camera/radar belt — instead of the distributed corner-roof lidars in the reference set. Height buys an unobstructed horizon and clean 360° overlap; a private owner would never tolerate the look, a fleet operator doesn't care.

So what actually varies vs. the reference set?

  • Roof 360° lidarThe reference car hides lidars in the roof corners; a robotaxi puts a dominant roof-top dome / spinning unit up high for unobstructed 360° long range. This is the headline change you asked about.
  • More, bigger sensorsNo styling or BOM ceiling. Sensor counts go up, units get larger and higher-spec (e.g. 17 MP imagers), and cleaning/heating systems are added because uptime is revenue.
  • Redundancy philosophyReference set uses cross-modal redundancy ("SRR side backed by SRL side"). Mobileye pushes further to two fully independent driving subsystems; Waymo keeps complementary fusion but with deep overlap.
  • Imaging / 4D radarRobotaxis lean on high-resolution imaging radar as a near-lidar-grade modality — which is exactly what lets Mobileye plan to drop side & rear lidar over time.
  • Audio keptThe microphone line in the reference set isn't an oddity — Waymo ships External Audio Receivers for the same siren-localisation reason. Both validate that note.
  • Geo-constrained ODDRobotaxis run in HD-mapped, geofenced areas, so a sensor set can be tuned to a known operating domain rather than the open-world generality a private L4 car must handle.
  • No driver fallbackThere's no human to take over, so onboard redundancy (sensors, compute, backup steering/braking) is non-negotiable — versus a private car that can still hand back to a driver.

Bottom line

The reference sheet is a distributed, packaging-constrained L4 set for a car people own. A robotaxi keeps the same modalities but escapes the constraints: it elevates lidar to a roof dome, scales counts up, adds cleaning and full hardware redundancy, and — in Mobileye's case — splits sensing into two independent subsystems. Waymo = redundant fusion, sensor-maximal. Mobileye = vision-first with an independent radar+lidar twin and a roadmap thinning lidar toward the front. The microphones and IMU triplet in your sheet line up cleanly with what both deploy.
